El programa tac

Se pide implementar una versión simplificada de la utilidad tac de Unix.

tac acepta como único argumento un nombre de archivo, e imprime por salida estándar las líneas del archivo en orden inverso.

Por ejemplo, si tenemos un archivo llamado nombres.txt con los nombres “Alicia”, “Bruno”, “Carolina” y “Daniel”, uno por línea y en ese orden, tac imprime: 

$ ./tac nombres.txt
Daniel
Carolina
Bruno
Alicia

Nota: en estos ejemplos, el símbolo $ no es parte del comando. Se usa para indicar que el texto que sigue es el comando que se debe escribir en la terminal. Por tanto, para ejecutar el comando sólo se debe escribir: ./tac nombres.txt↩︎

Observaciones

  1. Se puede asumir que el tamaño del archivo es menor que la cantidad de memoria disponible. Por tanto, se permite almacenar todo el archivo en memoria si fuere necesario (con complejidad espacial lineal).

  2. En caso de no existir el archivo pasado como argumento, el programa termina con código de salida 1, imprimiendo antes por salida de error estándar: 

    $ ./tac no_existe.txt
No se pudo abrir el archivo

  3. En caso de no recibir ningún argumento, o recibir más de uno, el programa también termina con código de salida 1, e imprime (también en stderr): 

    $ ./tac
Uso: ./tac <ARCHIVO>

    $ ./tac uno.txt dos.txt
Uso: ./tac <ARCHIVO>

Implementación

El programa se puede implementar en unas 50 líneas de C:

  1. verificar número de argumentos (CLA)
  2. abrir el archivo y leer todas las líneas
  3. imprimir las líneas en orden inverso
  4. liberar memoria

Para la lectura del archivo, se recomienda usar la función getline(): 

ssize_t getline(char **buffer, size_t *capacidad, FILE *archivo);

La principal ventaja de getline() es que automáticamente reserva la memoria dinámica necesaria para almacenar la línea: no se necesita llamar a malloc() a mano, ni preocuparse si la línea “entrará” en el buffer.

Sí se necesita, no obstante, liberar memoria al terminar: getline() llama a malloc() pero transfiere el “ownership” de la memoria al usuario.

La reserva de memoria dinámica se consigue apuntando cada vez el parámetro buffer a NULL, y capacidad a 0: 

char *linea = NULL;
size_t capacidad = 0;

if (getline(&linea, &capacidad, archivo) >= 0) {
  // Se leyó la siguiente línea del archivo, incluyendo el '\n' final.
}
free(linea);  // Se debe liberar.

La función getline() se encuentra definida en stdio.h. Como es una función de POSIX.1-2008, hay que declarar un identificador para indicar que lo queremos usar: 

// Al principio de tail.c y tac.c (antes de otros include):
#define _POSIX_C_SOURCE 200809L
#include <stdio.h>
#include <string.h>

El programa tail

Se pide implementar una versión simplificada de la utilidad tail de Unix. tail imprime las últimas N lineas de un archivo, o de entrada estándar.

En nuestro caso, el programa no acepta nombre de archivo y siempre trabaja con entrada estándar. Nuestra implementación acepta un número entero N como único argumento. Por ejemplo, usando el mismo archivo nombres.txt de antes: 

$ cat nombres.txt | ./tail 2
Carolina
Daniel

$ cat nombres.txt | ./tail 1
Daniel

$ cat nombres.txt | ./tail 4
Alicia
Bruno
Carolina
Daniel

$ cat nombres.txt | ./tail 100
Alicia
Bruno
Carolina
Daniel

Observaciones

  1. No se permite almacenar todo el archivo en memoria. Se puede almacenar, como máximo, N líneas. Tampoco se permite el uso de archivos temporales.

    Cabe notar que, al tratarse de entrada estándar, en ningún caso puede ser leída dos veces para averiguar el número de líneas primero.

  2. La función getline() también sirve para leer de entrada estándar, usando la constante stdin como archivo.

  3. En caso de recibir como argumento un número menor que 0, el programa termina con código de salida 1, imprimiendo antes por salida de error estándar: 

    $ ./tail -10
El número de líneas no puede ser negativo.

  4. El caso de número de argumentos incorrectos se maneja igual que para tac. Además, se imprime el mismo error si el argumento no es enteramente numérico: 

    $ ./tail abc.txt
Uso: ./tail <N>

    $ ./tail 12xyz
Uso: ./tail <N>

Archivo de dependencias

Se debe incluir un archivo deps.mk para indicar al corrector automático cómo compilar la entrega.

En particular, se debe indicar qué TDA emplea cada programa.

Por ejemplo, si tac usara un hash y tail usase un árbol, el archivo deps.mk consistiría de las siguientes dos líneas: 

# deps.mk: cada línea indica de qué OBJETOS se compone el programa.
tac: tac.o hash.o
tail: tail.o arbol.o

En el caso de haber creado una biblioteca de funcionalidad común, lib.c, se debería indicar también lib.o como dependencia: 

tac: tac.o lib.o hash.o
tail: tail.o lib.o arbol.o

Para la compilación, pueden invocar a gcc a mano como hasta ahora o, de manera opcional, emplear el siguiente Makefile: 

# Flags de gcc. Se usan automáticamente para compilar *.c → *.o
CFLAGS := -g -std=c99 -Wall -Wformat=2
CFLAGS += -Wshadow -Wpointer-arith -Wunreachable-code
CFLAGS += -Wconversion -Wno-sign-conversion -Wbad-function-cast

# Se pueden comentar durante el desarrollo.
CFLAGS += -Werror

# Si se ejecuta `make` a secas, se compilarán tac, tail y strutil.
#
# También se puede invocar `make tac`, `make tail` o `make strutil.o` para
# compilar una parte del TP solamente.

all: tac tail strutil.o

include deps.mk

Subcadenas con substr()

La función substr() permite obtener un prefijo de longitud N de una cadena dada.

Por ejemplo, la llamada substr("Hola mundo", 6) devolvería la cadena “Hola m”.

Documentación: 

/*
 * Devuelve una nueva cadena, allocada dinámicamente, con los primeros ‘n’
 * caracteres de la cadena ‘str’. El caller toma ownership de la cadena
 * devuelta.
 *
 * Devuelve NULL si no se pudo allocar memoria.
 */
char *substr(const char *str, size_t n);

Ayuda: usar la función strncpy().

Arreglos dinámicos de cadenas

split()

La función split() divide una cadena en subcadenas en cada ocurrencia de un caracter de separación determinado. Por ejemplo, separando por comas: 

split("abc,def,ghi", ',')  →  ["abc", "def", "ghi"]

En C, devolveremos el resultado como un arreglo dinámico de cadenas dinámicas terminado en NULL. Esto es: 

// Ejemplo de arreglo dinámico de cadenas
char **strv = malloc(sizeof(char*) * 4);
strv[0] = strdup("abc");
strv[1] = strdup("def");
strv[2] = strdup("ghi");
strv[3] = NULL;

El prototipo y la documentación de split() queda en: 

/*
 * Devuelve en un arreglo dinámico terminado en NULL todos los subsegmentos de
 * ‘str’ separados por el carácter ‘sep’. Tanto el arreglo devuelto como las
 * cadenas que contiene son allocadas dinámicamente.
 *
 * El caller toma ownership del vector devuelto. La función devuelve NULL si
 * falló alguna llamada a malloc(), o si ‘sep’ es '\0'.
 */
char **split(const char *str, char sep);

Casos borde: 

split("abc,,def", ',')  →  ["abc", "", "def"]
split("abc,def,", ',')  →  ["abc", "def", ""]
split(",abc,def", ',')  →  ["", "abc", "def"]

split("", ',')  →  [""]
split(",", ',') →  ["", ""]

Ayuda: usar la función strchr() para encontrar el separador, y la propia función substr() para realizar la copia.

join()

La función join() es la inversa de split(). Concatena un arreglo de cadenas terminado en NULL mediante un caracter de separación: 

// Ejemplo de uso de join
char **strv = split("abc,def,ghi", ',');
char *resultado = join(strv, ';');  // "abc;def;ghi"

char **palabras = split("Hola mundo", ' ');
char *otro_resultado = join(palabras, ',');  // "Hola,mundo"

El prototipo y documentación es: 

/*
 * Devuelve una cadena, allocada dinámicamente, resultado de unir todas las
 * cadenas del arreglo terminado en NULL ‘strv’.
 *
 * El caller toma ownership de la cadena devuelta. La función devuelve NULL
 * si no se pudo allocar memoria.
 */
char *join(char **strv, char sep);

Casos borde: 

join([])        →  ""
join([""])      →  ""
join(["abc"])   →  "abc"
join(["", ""])  →  ","

Ayuda: usar la función stpcpy() para concatenar cadenas. (Necesita la macro _POSIX_C_SOURCE antes de incluir string.h, como se explicó, anteriormente.) Acá pueden encontrar un ejemplo de uso.

Complejidad algorítmica: se espera que la función tenga complejidad \(\mathcal{O}(n)\) tanto espacial como temporal (con \(n =\) longitud de la cadena resultante).

free_strv()

free_strv() libera la memoria asociada con un arreglo dinámico de cadenas dinámicas: 

/*
 * Libera un arreglo dinámico de cadenas, y todas las cadenas que contiene.
 */
void free_strv(char *strv[]);

Ayuda con Valgrind

El uso de Valgrind con tac y tail tiene la particularidad de que los programas toman argumentos y leen y escriben de entrada y salida estándar.

El paso de argumentos es trivial, se añaden a la línea de comandos al final: 

$ valgrind --leak-check=full --track-origins=yes ./tac nombres.txt
                                                       ^^^^^^^^^^^
$ valgrind --leak-check=full --track-origins=yes ./tail 10
                                                        ^^

En el caso de tail, sin embargo, hay que permitir que lea de entrada estándar; pero se realiza de igual manera que en el ejemplo: 

$ cat nombres.txt | valgrind --leak-check=full --track-origins=yes ./tail 2
  ^^^^^^^^^^^^^^^

Por último, si lo que imprimen por salida estándar tac o tail resulta molesto para leer los mensajes de Valgrind, se puede eliminar para una ejecución añadiendo >/dev/null al final. Por ejemplo: 

$ valgrind --leak-check=full --track-origins=yes ./tac nombres.txt >/dev/null
                                                                   ^^^^^^^^^^

$ cat nombres.txt | valgrind --leak-check=full --track-origins=yes ./tail 2 >/dev/null
                                                                            ^^^^^^^^^^
